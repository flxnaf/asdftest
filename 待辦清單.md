## 4. 文字內容（Copy）— 必須核實或填寫

### 4.1 首頁副標題（Hero subhead）

**檔案：** `src/i18n/dict.ts` → `hero.sub` (EN + ZH)

目前文案：
> 「A Line Technologies 是亞太區隱形牙箍實驗室、牙科診所及新興品牌背後的治療方案設計夥伴，由矯齒科醫生主導⋯⋯」

**需確認：** 「亞太區」的覆蓋範圍是否屬實？如未有實際合作夥伴，建議改為更穩妥的措辭（例如「於香港成立、面向亞太區的⋯⋯」）。

### 4.2 臨床團隊資料（Team）

**檔案：** `src/i18n/dict.ts` → `team.members`（共 3 位）

目前以方括號標示需填寫的位置：
- `[臨床總監姓名] 醫生` → 換成真實姓名
- `[治療規劃主管姓名] 醫生` → 換成真實姓名
- `[資深顧問姓名] 醫生` → 換成真實姓名

每位均需填寫：
- `name` — 姓名
- `role` — 職位
- `creds` — 學歷及資歷（如 BDS、MOrth、年資）
- `body` — 簡短介紹
- `initials` — 圓形頭像中顯示的兩字英文縮寫

### 4.3 合規與資訊保安聲明（Compliance promises）

**檔案：** `src/i18n/dict.ts` → `compliance.promises`（共 6 項）

每一項都是對外承諾，必須真實。請逐項核實：

| 承諾 | 須確認的事項 |
|---|---|
| 「靜態 AES-256 加密，傳輸 TLS 1.3」 | 實際使用的加密規格是否如此？ |
| 「每個個案皆有審計記錄」 | 是否有完整審計系統？ |
| 「可提供 DPA 及 BAA」 | 是否已備有標準範本可供合作夥伴簽署？ |
| 「必經矯齒科醫生簽核」 | 流程上是否如此？ |
| 「可選擇亞太及歐洲寄存地點」 | 雲端服務是否真有此選項？ |
| 「每年由外部公司進行滲透測試」 | 是否真的有此安排，或計劃中？ |

**重要：** 不能擔保的項目請刪除。在受規管的牙科科技領域，誇大合規聲明屬法律風險。

## 部署備忘

- 網站於 Vercel 自動部署：每次 `git push` 到 `main` 分支後自動建置。
- 修改檔案後流程：
  1. 在本機修改
  2. `npm run build`（核實建置成功）
  3. `git add .` → `git commit -m "..."` → `git push`
  4. 等待 Vercel 完成部署（通常 1–2 分鐘）
